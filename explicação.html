Aqui será explicado como executar os conceitos do outro site:

Controle de Versão e de mudanças: Para o controle de versão, se deve criar uma pasta, dentro dessa pasta devem haver outras três pastas, uma delas é a pasta principal, que possui o arquivo completo em sua versão atual, outra pasta possui arquivos pesados, como arquivos de imagem, som, ou binários, que serão acessados pela pasta principal, a terceira pasta é a pasta de versões anteriores, aqui estarão marcados via data as pastas de cada versão anterior, mas apenas da pasta principal, para evitar o consumo que a pasta de arquivos pesados teria, essas pastas devem ser marcadas com data e hora de sua criação, cada modificação será mostrada, através de um bloco de notas, assim os desenvolvedores podem saber qual deles fez que modificação.

Integração Contínua: Para a integração contínua, os desenvolvedores precisam de um planejamento de datas para quando cada um deles deve mandar uma atualização em seus arquivos, essas atualizações serão mandadas para o repositório, assim, não deve haver muito tempo entre as atualizações de cada desenvolvedor, e tudo pode ser testado a qualquer momento, e não haverá problemas de espera de entrega de arquivos ou incompatibilidade quando o projeto for finalizado.
